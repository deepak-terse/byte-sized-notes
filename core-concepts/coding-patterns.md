# Coding Patterns

## Design Pattern:

### Creational
- Factory Method: abstraction
- Singleton: ensures single instance
- Builder

### Structural
- Adapter
- Decorator: modify the functionality at run-time
- Facade: an object that act as a front facing interface masking more complex underlying code 

### Behavioral
- Strategy
- Observer: subject notifies all dependents on any state change
- State: changes behaviour if the internal state changes

## Principles:

### SOLID:

5 principles of OOPs that make software more understandable, flexible, and maintainable.
- Single Responsibility: a class to have only 1 responsibility
- Open-Closed: open for extension but closed for modification
- Liskov Substitution: obj of superclass should be substitutable with obj of subclass
- Interface Segregation: no compulsion to implement the interface not in use
- Dependency Inversion: high level modules should be less dependent on low level modules
